01 Principio rector

Reglas que guían el wizard

  1. El wizard puebla TODA la verdad fiscal disponible sin preguntar · incluido el cruzado entre años (cambios de inquilino · cambios de uso · fechas inferidas)
  2. Solo se pregunta lo que el cliente tiene fresco (datos personales · empleador actual · IBANs activos · inmuebles que reconoce). Lo demás se rellena automáticamente.
  3. Datos del XML = verdad consumada · ATLAS no juzga · no avisa · no recalcula. Si Hacienda dice X · ATLAS dice X.
  4. El usuario enriquece después desde los módulos correspondientes · no se le obliga a completar en el momento de la importación.
  5. Reutilizar wizards existentes (nómina · autónomos · venta inmueble) con pre-relleno y opt-in · no diseñar wizards nuevos.
  6. Patrón común · paso propio detectado en el wizard · pre-relleno + check "¿quieres crear?" · saltable.

Sub-problemas a resolver en la sesión propia

  • Renaming · "sin_identificar" pierde sentido si tiene NIF · revisar para Tipo 1 vs Tipo 2 (sin_identificar es válido y necesario para T48 · pero hoy se aplica también a casos con DNI)
  • Entidad "Bote anual sin identificar" · nueva · store nuevo · estados (pendiente_total/parcial/cerrado/sobre_asignado)
  • Asignación retrospectiva · contratos reales que el usuario crea van descontando saldo del bote
  • Caso mixto · UX que permita Contract LAU + Bote NO LAU en mismo inmueble sin pisarse · cálculo de proración gastos
  • Cambios entre años detectables del XML · cierre automático de Contract LAU al cambiar NIF · creación nuevo Contract con fecha nueva
  • Datos del XML que hoy se pierden y deberían usarse · fechaContrato · TAR1/TAR2 · reducción art.23.2 · nifsDetectados
  • Modelo Contract V2 · añadir alcance · habitacionNumero · habitacionColor · tipoAlquiler (T5 mergeó UI pero NO modelo)
  • Vista "Sin identificar" en /contratos · UX de completar/asignar

Datos fiscales AEAT (capturas) · sesión propia futura
foco aparte

Distintos del XML · son las capturas pantallazo de los datos fiscales que la AEAT pone a disposición antes de la campaña Renta. NO exportables · solo imágenes. Se tratará cuando trabajemos importación de datos fiscales.

Lo que aportan que el XML no

  • Cuentas bancarias con intereses (IBANs adicionales · entidad · intereses · retenciones)
  • Préstamo hipotecario vivienda habitual (solo vivienda habitual · NO inversión)
  • Modelos M303 · M130 · M390 presentados
  • Crypto histórico completo reportado por exchanges
  • Cuotas asociaciones / colegios profesionales
  • Inmuebles que Hacienda sabe que tienes (incluyendo uso propio)
  • Compra/venta valores (M198)
